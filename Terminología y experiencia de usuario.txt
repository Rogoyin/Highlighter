# Terminología.

# Entidades.
# - Client: es la aplicación que va a operar sobre las N_Accounts.
# - Token: es un código de autenticación que tu aplicación usa para acceder a la API de Notion y realizar operaciones.
# - Block: es la entidad que contiene los datos del Highlight, Author, Book, Date, y demás. Es un bloque. Cada Block es una fila del df.
# - Highlight: es la frase extraída del Book, es decir, la frase recortada, y hay una por cada Block.
# - Author: el autor que escribió el Highlight.
# - Book: el libro del que se extrajo el Highlight.
# - Date: la fecha en que se hizo el subrayado.
# - Phrase: es la frase en el libro, que luego se extrae y se transforma en Highlight.
# - F_Block: Blocks en los cuáles se opera y tienen C_Blocks.
# - C_Block: surgen de las operaciones que se hacen en los F_Blocks.
# - Clippings: grupos de Blocks.
# - N_Block: bloques de Notion. Es una unidad específica dentro de las N_Page.
# - N_Page: página de Notion. Es una entidad más amplia que los N_Blocks.
# - N_Match: el conjunto o dúo, la conexión Clipping-N_Page, producto de un Match.
# - N_Account: cuenta de Notion.
# - Sub_Line: línea por debajo del Highlight cuando se exporta a archivo, que está compuesta por Author, Book o ambos.

# Acciones.
# - Cut: es la acción de subrayar una frase y transformarla en Highlight.
# - B_Sort: acción de reordenar, agrupar, dividir y/o unir Blocks y transformar el O_Order en el F_Order.
# - H_Processing: panel de procesamiento del contenido de los Highlights.
# - Paste: pegado de los Blocks en el formato de salida.
# - Match: ligar N_Page con Clipping.
# - N_Ubication: el lugar específico (N_Page y N_Block) dónde se va a Paste cada Clipping.

# Objetos.
# - File: es el .txt que expide el Kindle y el usuario adjunta a la web. Es de dónde se van a extraer todos los datos.
# - H_Database: es la database principal. Cada fila es un Block, y cada columna es un atributo del Block.
# - N_Database: es la N_Page principal dónde están las N_Pages hijas dónde se van a pegar los Highlights.
# - Explorer: explorador de archivos de la computadora.
# - N_Explorer: explorador de páginas de Notion.

# Página.
# - P_Start: panel de inicio.
# - P_Display: panel de representación visual de los Blocks, dónde puede realizarse el B_Sort.
# - P_Process: panel dónde se realiza el H_Processing.
# - P_Notion: panel dónde se elige el formato de salida de los Blocks, organizado por Clipping, en Notion.
# - P_End: panel donde se hace el Paste final.
# - O_Order: el orden establecido de los Blocks en el File.
# - F_Order: el orden final de Blocks elegido por el usuario.
